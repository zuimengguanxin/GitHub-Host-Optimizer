# GitHub Host 智能优选与自动维护工具 - 完整需求文档（含三层兜底+可配置+测速优选）
## 一、项目概述
### 1.1 项目背景
GitHub 访问依赖 hosts 文件配置域名与 IP 映射，公开 Host 源（如 520host、ineo6）IP 稳定性、速度差异大，手动切换、测速、维护效率极低；极端场景下公开 Host 源全部失效时无兜底方案，导致 GitHub 无法访问。本项目旨在开发一款 Windows 端 PowerShell 自动化工具，实现**主 Host 全量检测、IP 测速优选、智能故障替换、多层兜底防护、全配置化管理**，彻底解决 GitHub Host 手动维护痛点。

### 1.2 核心目标
1. 全自动检测用户维护的主 Host，基于 TCP 443 端口测速筛选**延迟最低、连通性最优**的 IP；
2. 实现**被动式备用池更新**（仅极端故障触发），减少无效网络请求；
3. 搭建**三层兜底防护体系**（主 Host → 备用池 → 第三方 DNS/IP 查询 API），覆盖所有故障场景；
4. 支持**全配置化自定义**，用户可自由增删备用 Host 源、第三方 IP 查询服务；
5. 保证工具稳定性，任何异常场景均保留原配置，不破坏系统 hosts 可用性。

### 1.3 运行环境
- 操作系统：Windows 10/11（x64）
- 运行环境：PowerShell 5.1 及以上版本
- 权限要求：必须以**管理员身份**运行（支持系统 hosts 文件写入）

## 二、核心概念与数据源定义
### 2.1 核心数据源（全可配置）
1. **主 Host 文件（用户维护）**
    - 定义：用户自定义的 GitHub 核心域名+IP 配置文本文件（默认：`my-github-hosts.txt`）；
    - 包含域名：github.com、api.github.com、raw.githubusercontent.com、gist.github.com、avatars.githubusercontent.com、assets-cdn.github.com 等 GitHub 关键域名；
    - 优先级：最高，所有检测、替换流程均以主 Host 为核心处理对象。

2. **备用 IP 池（本地缓存）**
    - 定义：从远程公开 Host 源拉取的域名-IP 映射集合，本地内存缓存，不主动更新；
    - 数据来源：用户配置的多个远程公开 Host 源（如 520host、ineo6、gitmirror 等）；
    - 处理规则：拉取时自动去重，避免重复 IP 冗余测试。

3. **远程备用 Host 源（可配置）**
    - 定义：网络上公开的、持续维护的 GitHub Host 仓库地址，作为备用 IP 池的更新来源；
    - 特性：支持用户自由增删、修改源地址，默认内置 3 个稳定源，可自定义扩展。

4. **第三方 IP 查询 API（终极兜底，可配置）**
    - 定义：公共 DNS 解析/IP 查询服务接口，通过域名实时获取最新公网 IP；
    - 作用：当主 Host + 备用池全部失效时，作为最后一层兜底，获取全新 IP 候选；
    - 特性：支持配置多个 API 互相兜底，避免单一服务失效导致工具不可用。

### 2.2 核心指标定义
1. **连通性**：TCP 443 端口（GitHub HTTPS 端口）连接成功，判定为 IP 可用；
2. **延迟（ms）**：TCP 443 端口握手耗时，耗时越低代表访问速度越快；
3. **最优 IP**：同一域名下，**连通性正常 + 延迟最低**的 IP。

## 三、核心功能需求
### 3.1 基础配置管理功能
1. **配置文件化管理**
    - 提供独立配置模块（支持 JSON/变量配置），用户可自定义：
      - 主 Host 文件路径、系统 hosts 写入路径；
      - 远程备用 Host 源列表（增删改任意地址）；
      - 第三方 IP 查询 API 列表（支持多接口配置）；
      - 测速超时时间、重试次数等参数。
2. **配置容错**
    - 配置项缺失时，自动使用默认配置；
    - 无效配置（如错误 URL）自动跳过，不中断工具运行。

### 3.2 主 Host 全量检测与测速功能
1. **强制全量检测**
    - 工具**每次运行必须逐条检测主 Host 中所有 GitHub 域名**，无跳过、无定时、无例外；
    - 检测内容：每条记录的 IP 连通性 + 连接延迟（ms）。
2. **精准测速逻辑**
    - 采用 TCP 443 端口握手测试（替代 ping），贴合 GitHub 实际访问场景；
    - 记录每个可用 IP 的延迟，自动排序生成测速排行榜；
    - 超时时间可配置（默认 1000ms），超时判定为 IP 不可用。
3. **结果实时展示**
    - 实时输出每条域名的测试状态：正常（含延迟）、失效、测速中；
    - 展示同一域名下所有候选 IP 的延迟排序，明确最优 IP。

### 3.3 智能 IP 优选与替换（核心流程）
对主 Host 中**单条域名+IP 记录**，按**三层优先级+测速优选**逻辑处理，流程如下：

#### 第一层：主 IP 优选
1. 测速用户配置的主 IP；
2. 若主 IP 连通正常：直接判定为当前域名的**最优 IP**，保留原记录，不触发后续流程。

#### 第二层：本地备用池优选
1. 主 IP 不可用时，遍历**本地缓存备用池**中**同域名的所有 IP**；
2. 对所有候选 IP 逐一测速，筛选连通正常的 IP；
3. 若存在可用 IP：按延迟排序选**最优 IP**，替换主 Host 中的失效 IP，标注替换记录（原 IP+替换时间），**不更新备用池**。

#### 第三层：备用池更新+重试
1. 主 IP + 本地备用池同域名所有 IP **全部不可用**时，**唯一触发远程备用池更新**；
2. 拉取用户配置的所有远程备用 Host 源，合并去重生成**新备用池**；
3. 对新备用池同域名 IP 重新测速，筛选最优 IP；
4. 若存在可用 IP：替换主 Host 失效 IP；
5. 若新备用池仍无可用 IP：进入**终极兜底流程**。

#### 第四层：第三方 API 终极兜底
1. 调用用户配置的**所有第三方 IP 查询 API**，通过域名实时解析最新公网 IP；
2. 对解析出的所有 IP 去重、测速，筛选最优 IP；
3. 若存在可用 IP：替换主 Host 失效 IP；
4. 若所有 API 解析失败/IP 全不可用：**保留用户原主 IP，不做任何修改**，输出无可用 IP 警告。

### 3.4 备用池管理功能
1. **被动更新机制**
    - 备用池无定时更新、无主动拉取，**仅满足“同域名主 IP+本地备用 IP 全失效”** 时触发更新；
    - 每次更新仅拉取配置的远程备用源，合并去重后覆盖本地缓存。
2. **多源兼容与容错**
    - 支持同时拉取多个远程备用源，单个源拉取失败时自动跳过，不影响整体流程；
    - 备用池更新失败时，沿用旧缓存备用池，保证工具可用。

### 3.5 系统 Hosts 写入功能
1. **自动写入**：检测、优选、替换完成后，自动将最终有效配置写入系统 hosts 文件（`C:\Windows\System32\drivers\etc\hosts`）；
2. **格式保留**：保留主 Host 中的注释、空行、自定义格式，仅替换失效 IP 条目，不破坏文件可读性；
3. **安全写入**：写入前备份原系统 hosts 配置（可选），避免误修改导致系统异常。

### 3.6 状态提示与日志功能
1. **实时进度提示**
    - 关键节点提示：备用池更新触发、第三方 API 调用、IP 替换、兜底触发等；
    - 结果提示：每条域名的最终状态（正常/已替换/无可用 IP）、最优 IP 延迟。
2. **错误警告**
    - 远程源拉取失败、API 调用失败、无可用 IP 等异常场景，输出红色警告信息；
    - 明确告知用户异常原因与兜底处理结果。
3. **简易日志**
    - 记录每次运行的核心操作：检测时间、替换条目、备用池更新记录、兜底触发记录，便于问题排查。

## 四、非功能需求
### 4.1 性能需求
1. **检测效率**：仅测试必要 IP（主 IP+失效域名的候选 IP），无冗余测试，单次运行耗时≤15 秒；
2. **网络占用**：日常运行仅测速本地 IP，无额外网络消耗；仅极端故障时拉取远程源/调用 API，最小化网络流量。

### 4.2 稳定性需求
1. **容错机制**
    - 远程备用源、第三方 API 失效时，自动降级，不中断主流程；
    - 测速超时、网络波动时，自动重试 1 次，重试失败判定为不可用。
2. **兜底保护**
    - 任何异常场景（全层失效、配置错误、权限不足），均**保留用户原主 IP 与系统 hosts 配置**，绝不删除、清空关键配置；
    - 无管理员权限时，禁止写入系统 hosts，仅输出检测结果，避免权限报错。

### 4.3 易用性需求
1. **零学习成本**：用户仅需维护主 Host 文件，无需理解底层逻辑，双击即可运行；
2. **配置简洁**：配置项清晰易懂，支持直接修改脚本变量完成自定义，无需复杂配置文件。

### 4.4 兼容性需求
1. 兼容 Windows 10/11 全版本 PowerShell 环境；
2. 兼容国内网络环境，适配代理、直连等多种网络场景。

## 五、边界场景与异常处理（全覆盖）
| 场景 | 处理逻辑 |
|------|----------|
| 主 IP 连通正常 | 保留原 IP，无额外操作 |
| 主 IP 失效+本地备用有可用 IP | 测速选最优，替换原 IP，不更新备用池 |
| 主 IP+本地备用全失效 | 更新备用池→测速→替换；仍失效则触发第三方 API 兜底 |
| 备用池更新失败（远程源全挂） | 沿用旧备用池，输出警告，继续流程 |
| 第三方 API 全失效/解析 IP 全不可用 | 保留原主 IP，输出无可用 IP 警告 |
| 无管理员权限 | 仅输出检测结果，禁止写入系统 hosts |
| 主 Host 文件缺失 | 自动创建默认主 Host 模板，提示用户补充配置 |
